# GitHub Milestone: v0.2.0

## Title

v0.2.0 - Prompt Layers And Local Memory Foundations

## Goal

Establish the first true DualMind Studio architecture layer beyond raw browser orchestration.

## Scope

### Must Have

- Prompt Blueprint shared types
- prompt composition helpers in the engine
- local memory schema and persistence primitives
- memory-aware resume behavior
- advanced setup UI for identity, style, and memory controls

### Should Have

- transparent memory preview in the UI
- memory clear/prune controls per session
- improved agent documentation in the README

### Not Yet

- long-term cross-project memory
- cloud sync
- generic multi-provider plugin system

## Suggested Tracking Issues

1. Prompt blueprint and layered composer
2. Session memory schema and persistence
3. Memory-aware discussion resume
4. Advanced setup panel for identity/style/memory
5. Memory transparency and controls in UI

## Exit Criteria

- the engine no longer relies on flat role prompts alone
- a session can store and reuse structured memory safely
- users can inspect and clear memory-related state
- the UI explains identity/style/memory clearly enough for advanced users
